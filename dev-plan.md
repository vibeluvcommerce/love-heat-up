# Love Heat Up 一个人从0到1的开发计划  
目标：一个人（你）每天都能在浏览器里看到明显进展，绝不憋大招，平均每天工作8-12小时，第45天做出可上线、可赚钱、让人尖叫的完整版本。

**总里程碑**  

- 第6天 → 单人完整脱衣大富翁（核心爽点100%达成）  
- 第9天 → 能挂机看3个AI辣妹互相脱光  
- 第15天 → 4人异地实时联机稳定  
- 第21天 → 欲望值♥+三张最骚特殊卡  
- 第30天 → 全部最终立绘、语音、CG替换完成  
- 第45天 → PWA打包上线，可赚钱

### 第0天（环境日，4小时）
- pnpm create vite@latest client -- --template vue-ts
- pnpm create vite@latest → 选 vue-ts
- cd client → pnpm add phaser howler @types/howler vite-plugin-pwa
- mkdir server → cd server → python -m venv venv → pip install flask flask-socketio eventlet
- 同时跑通 vite dev 和 flask run，能看到两个终端都活着

### 第1天 → Phaser3彻底跑通
- 在 client/src/views/GameView.vue 放 <canvas id="game-canvas" />
- 新建 src/game/MainGame.ts → new Phaser.Game(config) 挂载到上面canvas
- 创建 BootScene → preload 一张辣妹占位立绘 → create 里 add.image(540,960)
- 加上键盘左右键让立绘能左右移动 50px
- 晚上效果：打开游戏页，看到一张大辣妹能在屏幕上左右走

### 第2天 → 心形路径 + 骰子移动
- 用 Phaser3 Graphics 画 40 个小方块围成粗糙心形（或直接用 40 个 sprite 摆成心形）
- 写一个数组 pathPoints[40] = [{x,y}, ...]
- 加一个圆形棋子，点击“掷骰子”按钮 → 随机1-6 → 用 Phaser Tween 依次走到对应格子
- 晚上效果：点骰子，小人一格一格沿着心形走完

### 第3天 → 土地系统 + 买地
- 每格土地数据：{id, price:800-2000, owner:null, level:0}
- 走到未拥有格子 → Vue弹窗“是否购买XXX酒店？”
- 购买后格子变色 + 插小旗子 + 钱-价格
- 玩家初始金钱15000，右上角实时显示
- 晚上效果：一圈下来买了好几块地，钱在减少，有大富翁的感觉了

### 第4天 → 脱衣系统（当天最爽）
- 准备4个角色（玩家+3AI），每个角色先占6张图（哪怕同图改名）
- 全局状态 clothes: 5 → 0 （5=全衣，0=全裸）
- 给3个AI辣妹各分配4-5块“归属格”（写死在地图数据里）
- 当有人买下AI的归属格 → 对应AI clothes-- → 大立绘立刻切换 → camera.shake(300) + 播放脱衣布料音效
- 晚上效果：买某个格子 → 对应辣妹秒脱一件 + 屏幕震动 → 爽到起飞

### 第5天 → 过路费 + 娇喘 + 抖动
- 走到别人地 → 扣钱（价×(level+1)）
- 同时触发：
  - 地主大立绘用 tween 上下左右疯狂抖动 800ms
  - Howler播放随机一条娇喘（先放5条占位语音）
  - 屏幕红色vignette闪一下
- 晚上效果：踩到别人地 → 钱少 + 对方辣妹抖 + 娇喘 → 硬了

### 第6天 → 破产淘汰 + 胜者30s CG
- 钱<0 → clothes强制=0 → 切换倒地姿势图 → 播放10秒“挣扎”位移动画
- 场上剩1人 → 黑屏 → 播放30秒占位高潮视频（随便找一段）
- 结束后返回大厅按钮
- 晚上效果：把3个AI搞破产 → 看30秒赢家CG → 单人最小闭环彻底完成！（第6天就爽完）

### 第7天 → 回合制 + AI自动玩
- 写 TurnManager，玩家操作完 → 自动轮流执行3个AI回合
- 每个AI：90%概率买地，10%不动
- AI交费一样触发抖动+娇喘
- 晚上效果：点开始就能挂机看3个AI互相脱到全裸

### 第8-9天 → AI更聪明 + 小优化
- AI会升级自己的地
- 加简单表情气泡（“啊！”“不要！”）
- 加背景音乐循环
- 第9天晚上效果：完全可以挂着看AI辣妹互相玩到高潮，拿去钓群友已足够

### 第10天 → Flask-SocketIO基础通信
- server/app.py 写最简单房间：/create_room 返回 room_id
- client 用 socket.io-client 连接本地5000
- 加入房间成功 → 页面显示“玩家2已加入”
- 晚上效果：两个浏览器窗口互相看到对方加入

### 第11天 → 准备阶段同步
- 房间内4个玩家都有“准备”按钮
- 全员准备 → 服务器广播 start_game + 同一随机种子
- 晚上效果：4个人都按准备 → 同时进入游戏画面

### 第12天 → 掷骰子实时同步
- 轮到谁由服务器决定 → 广播 dice_roll {playerId, value}
- 所有客户端同时看到那个人掷骰子并移动
- 晚上效果：异地4个人，动作完全同步

### 第13天 → 买地、交费、脱衣实时同步
- 所有关键操作都 emit 到服务器 → 服务器再 broadcast
- 晚上效果：一个人买地，所有人都看到对应辣妹实时脱衣服

### 第14天 → 破产同步 + 胜者CG同步倒计时
- 有人破产 → 所有人看到她倒地挣扎
- 最后一人胜利 → 所有人同时进入30秒CG倒计时
- 第14天晚上：4人联机已基本可用

### 第15天 → 掉线重连 + 防作弊校验
- 所有随机数由服务器发
- 掉线30秒内可重连
- 第15天晚上：4人联机彻底稳定（95%的商业价值已达成）

### 第16-18天 → 大厅系统（纯Vue）
- 第16天：房间列表 + 创建房间 + 加入房间
- 第17天：游客UUID + 改昵名 +名 + 头像选择 + 准备按钮
- 第18天：快速匹配 + 密码房

### 第19-21天 → 欲望值♥ + 三张最骚卡
- 第19天：每回合自动+10♥，右上角显示
- 第20天：♥商店 + 第一张卡“强制脱衣”
- 第21天：第二张“10秒持续抖动+连发娇喘”，第三张“小高潮扣2000+尖叫特效”
- 第21天晚上：辣度再次翻倍

### 第22-30天 → 美术资源全部替换（每天换一个角色）
- 每天一个角色：6阶段差分立绘 + 8条娇喘 + 专属30s高潮CG
- 同时替换地图格子插图、BGM、音效
- 第30天晚上：所有占位资源全部替换为最终版，画面直接起飞

### 第31-35天 → 细节拉满
- 实时表情气泡（害羞/生气/高潮脸）
- 内置18+聊天表情包
- 胜者可以选择“被淘汰顺序”决定谁来侍奉
- 胜者视角CG多角度切换
- 移动端触控优化

### 第36-40天 → PWA + 打包优化
- 配置 vite-plugin-pwa，实现“添加到主屏幕”
- 资源分包、图片压缩、音效压缩
- 添加加载进度条

### 第41-45天 → 封测 + 修复 + 上线
- 拉100人内测群，收集bug和“想看更多大腿差分”需求
- 修复所有卡顿、音效重叠、联机掉帧
- 部署到 Vercel（前端）+ Railway / Render（后端）
- 第45天晚上：正式上线，可发群、可接广告、可卖新角色DLC
